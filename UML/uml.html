    • Unified Modeling Language

    Unified Modeling Language (UML) loodi ühise, semantiliselt ja süntaktiliselt rikka visuaalse modelleerimiskeele loomiseks keerukate tarkvarasüsteemide arhitektuuri,
    disaini ja juurutamise jaoks nii struktuurselt kui ka käitumuslikult. UML-il on ka rakendusi peale tarkvaraarenduse, näiteks protsessivoog tootmises.

    See on analoogne teistes valdkondades kasutatavate joonistega ja koosneb erinevat tüüpi diagrammidest.
    Kokkuvõttes kirjeldavad UML-diagrammid süsteemi ja selles olevate objektide piire, struktuuri ja käitumist.

    Structure Diagrams


    Package Diagram

    Package Diagrams näitavad teie süsteemi sõltuvusi ja neid kasutatakse peamiselt siis,
    kui teil on suur koodibaas. Paketid võivad sisaldada kasutusjuhtude diagramme,
    et illustreerida süsteemi funktsionaalsust ja visualiseerida tarkvarasüsteemi kihilist arhitektuuri.

    Class Diagram

    Objektorienteeritud programmeerimiskeeled põhinevad klassidel ja nendevahelistel suhetel.
    UML-i klassidiagrammide tähistus töötati välja nende programmide visuaalseks esitamiseks.
    Klassid sisaldavad nende atribuute (andmeid) ja käitumist (liikmefunktsioonid) ning on üksteisega seotud mitmesuguste konnektoritega.

    Deployment Diagram

    See on veel üks spetsiaalne diagramm suurte ja keeruliste süsteemide jaoks,
    kus tarkvara on juurutatud mitmesse süsteemi. Tänapäeval on veebiteenustel,
    nagu Amazon Web Services (AWS) ja Google Cloud Platform (GCP), oma tähistused - need kujundid leiate nende draw.io kujundite teegist.

    Object Diagram

    UML-i objektidiagrammid on klassidiagrammide erivorm. Nad visualiseerivad abstraktsemaid klassiskeeme nii,
    nagu oleksid need instantseeritud ja aitavad tarkvaraarendajatel kontrollida, kas nende abstraktsed struktuurid praktikas töötavad.

    Component Diagram

    Komponentide diagramm näitab, kuidas keeruka tarkvarasüsteemi erinevad komponendid on omavahel seotud ja suhtlevad liideste abil.
    Neid ei kasutata lihtsamate või sirgjooneliste süsteemide jaoks.

    Composite Structure Diagram

    Kui teil on keeruline tarkvarasüsteem, millel on palju erinevaid kasutusjuhtumeid,
    visualiseerivad UML-i liitstruktuuri diagrammid, mis programmi käivitamisel tegelikult juhtub.
    See esindab klassi sisemist struktuuri ja seda, kuidas see on seotud erinevate klassikomponentidega.

    Behavior Diagrams


    Activity Diagram

    Activity diagrams model the behavior of users and systems as they follow a process.
    They are a type of flow chart or workflow, but they use slightly different shapes.

    Use Case Diagram

    Use Case diagrammid luuakse, kui vaatate oma süsteemi või programmi nõudeid.
    Need esindavad funktsioone või tunnuseid, osalejaid ja nende omavahelist seost (nende suhteid).

    State Diagram

    State diagrammid on programmeerimisel kasutatud erinevate olekute kirjeldamiseks,
    milles süsteem võib olla aastakümneid. Need ei piirdu programmeerimisega - igal süsteemil ja protsessil on olekud.
    Neid diagramme on kiirem luua kui tegevus- või järjestusskeeme, kuid need näitavad vähem teavet.
    Olekudiagrammid näitavad, millised olekud viivad üksteiseni ja mis käivitab olekumuutuse.
